# System Map

## Purpose
Map the ecosystem of forces around a problem by identifying Drivers, Mainstay processes, and Enablers. The System Map reveals how systemic forces cascade through interconnected layers — showing why fixing one thing without understanding the whole system often creates new problems.

## Core Principle
"All work is process — a system involves People, Process, and Technology that share a common purpose. If each part focuses only on working efficiently in its own position, it leads to slower efficiency as a system."

## Structure

### Drivers (Top Layer)
Forces that exert pressure on the system. Drivers PUSH the mainstay to act.

Types of drivers:
- Policy decisions and regulatory mandates
- Financial pressures and funding models
- Market forces and demand patterns
- Resource availability (workforce, infrastructure)
- External constraints (geography, demographics)
- Insurance and coverage structures

For each driver, include:
- What the force is
- Specific data quantifying its impact
- How it connects to the mainstay processes (draw the arrows)

### Mainstay (Middle Layer)
The core processes that execute the drivers. These are the operational realities — what actually happens in the system day-to-day.

Types of mainstay elements:
- Service delivery mechanisms
- Access and scheduling systems
- Workforce deployment
- Coverage and eligibility processes
- Geographic and logistical operations

For each mainstay element, include:
- What the process is
- Current performance data
- Which drivers feed into it (multiple drivers often converge on one mainstay)
- How it connects to enablers below

### Enablers (Bottom Layer)
The structures, resources, and conditions that support the mainstay. Enablers can be positive (things that help) or negative (consequences that indicate system health).

Types of enablers:
- Infrastructure and facilities
- Funding and reimbursement mechanisms
- Technology and data systems
- Workforce training and capacity
- Customer/end-user behavior and response
- Trust and equity indicators

For each enabler, include:
- What it is
- Current state with data
- Which mainstay processes it supports or undermines
- Whether it's a positive enabler or a negative consequence

## Interconnections
The power of the System Map is in the arrows — the connections between layers. Key patterns to identify:

- **Multiple drivers → single mainstay**: Shows where pressure converges
- **Single driver → multiple mainstays**: Shows how one force ripples across the system  
- **Mainstay → negative enablers**: Shows where process failures create downstream consequences
- **Enabler feedback → drivers**: Shows reinforcing loops (vicious or virtuous cycles)


## Critical Rules
- If you don't understand the interrelatedness, your solution causes more problems
- Simple questions often require complex and reflective thinking if good solutions are to be found
- It is always better to manage by design than by default
- Drivers are often in conflict with each other — name the conflicts
- System maps live within a set of 1000 other system maps — acknowledge the boundaries
- One small change can have a huge impact on the system — identify the highest-leverage points
- Build insights from a CUSTOMER perspective — this reveals whether the organization is thinking about the customer

## How to Build
1. List out the drivers from an organization's perspective
2. Pick any driver and list the processes that execute it
3. Add more drivers and enablers as you list more processes
4. Map the connections between layers
5. Build insights from the customer perspective

## Questions to Ask the User
- "What are the main external forces or pressures acting on this system?"
- "What are the core day-to-day processes that make this system function?"
- "What resources, infrastructure, or conditions either help or hinder these processes?"
- "Are there reinforcing loops — situations where a problem makes itself worse?"
